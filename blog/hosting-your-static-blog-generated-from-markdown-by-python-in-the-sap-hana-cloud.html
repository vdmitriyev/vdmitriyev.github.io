Hosting Your Static Blog Generated from Markdown by Python in the SAP HANA Cloud

Published: March 2015
By Viktor Dmitriyev

In interesting.

tags: blog python pelican sap hana cloud html5

About

Current blog post is going to describe how to use mix from various technologies in order to build something interesing for fun.

The mixure of the technologies that are going to be using in this article are following:

  • Python 2.7
  • Pelican 3.5.0
    • This is of the the python packages that is able to generate static html pages from markdown
  • Markdown
    • Simple to write and read format, that became a kind of a standard for simple writing rotines
  • SAP HANA Cloud
    • The trial version will be use, if you don't have one, you can easily get it after easy registration
  • Git
    • You can install client if you don't have one from here
  • GitHub

Creating Empty HTML5 Application

After getting to the SAP HANA Cloud cockpit select from the main menu on the left "HTML5 Applications" (as it's shown on the image)

Then go and create new HTML5 application by selecting "New Application", as it's shown on the image

Give an name to the new application that you are creating, for instance I gave a simple name "blog", and after setting the name to a value of your choise, click on the save button

Congratualtions, you just created your own HTML5 application inside SAP HANA Cloud and you should see something similar to the image below. You cane to note, that you "only" created application and it's not running for a moment, to run it, you will need to set a version to your commit from git and only then you will be able to see rendering page in you browser. Later on I will show you how to do it.

Navigate to your newly created application by click on it's name. For a moment, you should see something similar to the image below. The image is telling you that the application is not yet activate, so let's more on and activate it.

After navigate to submenu titled "Versioning", you will see following important for us fields (see the image below) Git Repository Url and Actions

The urls that is shown to you under the title Git Repository Url will be used later on for the communication between you local git repository and remote repository hosted on SAP HANA Cloud.

Menu option Actions is very important to mane the SAP HANA Cloud understand which exactly version of the repositry to render. Without setting the action to active, the button that make all application run won't be event enabled.

Cloning Created HTML5 Application on Computer

After creating the first commit (that was actually created automatically by the system) you need to copy Git Repository Url and run command line utility (Windows: cmd, Linus: bash or similar). Before cloning git repository that was created inside SAP HANA Cloud, you need to have git client to be installed on your computer. You can downloed one from http://git-scm.com/. Open you terminal, navigate to the directory where you would like to clone recently created repository and type following command:

git clone <**Your Git Repository Url**>

You ternimal should look like on the image below

The git server will ask you to type in your credentials, you have to use the the same credentials you used to autorize yourself inside SAP HANA Cloud Cockpit (the same one your are using for scn.sap.com). Image below demonstrates how it should look like.

After creating git clone of your repository from SAP HANA Cloud, you can start working with it. Basically, you can upload any web related files (HTML, CSS,JS, etc.). For this particular example I used Python 2.7, it's Pelican 3.5.0 package and Markdown as a main text format for writing the blog posts. To process furher you will need to install Python 2.7, Pelican 3.5.0 and it's dependencies in order to be able to generate statis HTML files from markdown. Here you can find ready GitHub repository with all necessary configurations, scripts and detailed descriptions of it's usage. In order to merge two Git repositoties (GitHub and SAP HANA Cloud one) use a simple techniques - keep the SAP HANA Cloud's Git as a main one, and GitHub as a secondaty one. Here is how I did it (don't forget to open your terminal):

# clone the SAP HANA Cloud repository
git clone https://git.hanatrial.ondemand.com/<YOUR-SAP-HANA-CLOUD-USER-ID>/blog
cd blog

# adding additioanl git repostiry from github 
# the name of the repository is github
git remote add github https://github.com/vdmitriyev/saphanacloud-static-blog-example.git

# downloading repository from github
git pull -u github master

# uploading github repository to the SAP HANA Cloud
git push -u origin master

Setting a Version to One of Your Git Commits

If you will folllow order of git command above at the end you will be able to see at your SAP HANA Cloud's HTML5 Application the pre-generated html blog from the GitHub repository. However, the commited and pushed to the SAP HANA Cloud's git changes won't be available in the browser untill the time when you will specify the exact vesion of the commit to be published.

Navigate to the HTML5 application and select 'Versioning'. After the main working panel will load, click on the to the icon located in the 'Action' column and set desired vesion (as and example I set '0.0.1'), as it shown on the image below.

After setting the vesion you should see that the column 'Version' is poiting to the version you just set.

To activate your newly make version, click on the icon in the 'Action' column again and confirm your choice.

Now go to the "HTML5 Application Dashboard" and follow offered URL as it shown on the image below.

Congratulation! If you succeded in accomplish all the mentioned above stesps, you should some sort of blog as it shown below. Now you can keep writing your blog posts in markdown, compile them into static HTML with help of Python package Pelican and host your blog in the SAP HANA Cloud.
